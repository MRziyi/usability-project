# Phase 3D — Peer Comments Received

We received nine comments from classmates on our Phase 3C redesigns (posted on the Canvas forum, March 29 – April 20). This page summarizes the feedback and identifies key themes.

---

## Feedback Received

Nine reviewers commented: Stone Shi, Dua Shalan, Saira Gonzalez, Bashi Hariharan Yasodha, Giang Hoang, Zixiao Xie, Jingyuan Liang, Thiri Cynn, and Cindy Chung.

Full feedback is documented on the [Feedback](/feedback) page.

---

## Key Themes

### 1. Praise for website clarity and Problem Inventory structure
All nine reviewers noted the website was well-organized and easy to follow. Several specifically called out the severity ratings, design rationale, and side-by-side option comparisons as helpful.

### 2. Strong consensus on Option A for P1 (Upload file entry point)
Six reviewers (Stone, Bashi, Giang, Zixiao, Cindy, and Dua indirectly) preferred **Option A** — the prominent "Upload file" button — for the P1 Data Import redesign. This aligns with our own user study findings. One reviewer (Thiri) preferred Option C (Import Wizard) for stronger first-time scaffolding.

### 3. Data pills could clutter dense dashboards (P2)
Three reviewers (Giang, Zixiao, Cindy) independently flagged that persistent data pills or labels could create visual noise in complex, multi-chart dashboards. They suggested making them collapsible or color-coded.

### 4. Post-upload banner preferred for P2 (Dua)
Dua highlighted Option B (post-upload banner) as her favorite P2 solution — it intervenes at exactly the right moment to prevent the "upload = applied" misunderstanding.

### 5. Progressive disclosure for P4 (Dua)
Dua suggested showing basic editing controls by default and hiding advanced options behind a toggle. This addresses the tension between novice clarity and expert access without requiring separate UI modes.

### 6. Dual-mode UI suggestion (Saira)
Saira proposed the most architecturally significant suggestion: two separate UI layouts (selectable at login) — one for new users, one for experienced users.

### 7. Slides/PowerPoint-style text editing (Thiri)
Thiri (experienced with Tableau and PowerBI) preferred the drag-handle approach for P6, and suggested **double-click to enter edit mode** to prevent accidental text placement. She also preferred the panel redesign over right-click for P4.

### 8. Add original interface screenshots (Jingyuan)
Jingyuan suggested including screenshots of the current Looker Studio UI alongside each redesign proposal to make the "before" state clearer.

---

## Action Items → Phase 3E

- Confirm Option A for P1; evaluate incorporating Option C first-time guidance
- Address data pill clutter — collapsible / color-coded approach
- Evaluate Saira's dual-mode proposal vs. progressive disclosure as lighter alternative
- Add progressive disclosure to P4 panel
- Add double-click edit mode to P6 text annotation
- Respond to Jingyuan's screenshot suggestion with feasibility rationale
